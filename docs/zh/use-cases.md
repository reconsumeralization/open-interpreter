---
title: 用例
description: 常见任务，可交给 Open Interpreter 处理。
---

Open Interpreter 在您为其提供具体的工作空间、清晰的约束以及验证目标时表现最佳。

## 工程

- 根据复现步骤修复错误。
- 解释不熟悉的代码路径。
- 重构模块而不改变行为。
- 为回归问题添加测试。
- 升级依赖或 API 集成。
- 保持文档与代码更改同步。

## 审查与质量

- 审核 pull request 差异。
- 审计安全敏感代码。
- 对 CI 失败日志进行分流。
- 将不稳定测试报告转化为修复计划。

## 数据与文档

- 在一次性工作空间中分析 CSV 或日志导出。
- 根据源文件起草内部文档。
- 将 issue 讨论串转换为实现任务。

好的提示应包括目标、相关文件、约束以及能够证明工作完成的命令。
