Open Interpreter 的自定义是分层的。使用最小且持久的机制来满足需求。

| 机制 | 最适用 |
| --------- | -------- |
| `AGENTS.md` | 项目规则、命令、架构和约定。 |
| Config profiles | 模型、提供商、沙箱、批准和功能默认值。 |
| Skills | 可重复的工作流，包含说明、脚本和引用。 |
| MCP | 外部工具和数据源。 |
| Hooks | 确定性的策略、日志记录和验证脚本。 |
| Subagents | 用于调查或并行工作的专用辅助代理。 |
| Memories | 启用时的个人跨会话偏好。 |

## 推荐模式

1. 将稳定的仓库说明放在 `AGENTS.md` 中。
2. 将个人默认设置放在 `~/.openinterpreter/config.toml` 中。
3. 仅对受信任的仓库，在 `.openinterpreter/config.toml` 中放置项目特定的配置。
4. 对重复的工作流使用技能。
5. 对代理应直接调用的工具使用 MCP。
6. 对每次都应运行的确定性检查使用钩子。

## 保持指令简短

冗长的指令文件会使每个会话变得沉重。更倾向于使用简短的规则并链接到具体文件。将大型流程移入技能，以便仅在需要时加载。

## 审核自定义

自定义可能会改变运行的工具以及模型看到的上下文。信任新项目之前，请审核仓库本地的配置、技能、MCP 服务器和钩子。
